## WORKING OF PROGRAM

1. Multiple processes are arranged in a logical ring
2. A token is passed from one process to another
3. Only the process holding the token can enter critical section
4. After execution, token is passed to next process
5. This ensures mutual exclusion

---

## ALGORITHM STEPS

1. Initialize N processes in a ring
2. Generate a token at one process
3. Pass token sequentially (P1 → P2 → P3 → ... → Pn)
4. When process receives token:

   * If it wants to enter critical section → execute
   * Else → pass token to next process
5. Continue cycle

## WORKING FLOW

Token → Process 1 → Process 2 → Process 3 → ... → Process N → back to Process 1

---

## KEY CONCEPTS

1. Token:
   Special permission to enter critical section

2. Mutual Exclusion:
   Only one process executes critical section at a time

3. Ring Topology:
   Processes connected in circular form

4. Deadlock-Free:
   No two processes can hold token simultaneously

---

## IMPORTANT NOTES

1. Only one token exists in system
2. Token must always circulate
3. If token is lost → system fails
4. Ensures fairness (no starvation)
