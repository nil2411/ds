## WORKING OF PROGRAM

1. Server (master) starts
2. Clients (slaves) connect to server
3. Server requests time from all clients
4. Each client sends its local time
5. Server calculates average time
6. Server sends adjustment to each client
7. Clients update their clocks

---

## ALGORITHM STEPS (BERKELEY)

1. One node is selected as MASTER
2. Master sends request to all SLAVE nodes
3. Slaves send their current time
4. Master calculates average time
5. Master ignores faulty clocks (if any)
6. Master computes time difference
7. Master sends adjustment (not actual time)
8. All nodes update their clocks

👉 Master computes average time and synchronizes all clocks ([GeeksforGeeks][1])

## KEY CONCEPTS

1. Master Node:
   Controls synchronization

2. Slave Nodes:
   Send time to master

3. Clock Drift:
   Difference in system clocks

4. Synchronization:
   Making all clocks equal
